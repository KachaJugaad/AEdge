The Problem

Vehicles break down. Drivers find out too late.

Every year, unexpected vehicle failures cause accidents, stranded fleets, and millions in repair costs. Current telematics systems upload data to the cloud and send alerts minutes or hours later — if they send them at all.

⏳

Cloud Latency Kills

Traditional systems upload telemetry to a cloud server, run analysis, and push notifications back. Round trip: 2-30 seconds. An overheating engine doesn't wait 30 seconds. AnomEdge decides in under 100 milliseconds, on the vehicle.

🗻

Dead Zones Are Real

Mining sites, rural highways, construction zones — no cell signal means no cloud. Edge AI runs entirely offline. The vehicle's own hardware does the thinking. No internet? No problem.

💰

$15,000 Average Breakdown Cost

A single catastrophic engine failure costs $8K-$25K in parts, towing, and lost productivity. Catching a rising coolant temperature 60 seconds earlier turns a $15K engine replacement into a $200 coolant top-off.

👨‍🔧

Drivers Aren't Mechanics

Dashboard lights are cryptic. "Check Engine" tells you nothing. AnomEdge translates sensor data into plain English: "Engine overheating. Pull over within 5 minutes. Turn on the heater to draw heat."

📊

Alert Fatigue

Existing systems flood drivers with every minor fluctuation. AnomEdge uses a Trust Engine with cooldown & hysteresis — it only alerts when it matters, and it doesn't repeat itself for the same ongoing issue.

🏭

One Size Doesn't Fit All

A Ford F-450 and a Caterpillar 320 excavator have completely different signals and thresholds. AnomEdge has pluggable telematics adapters and policy YAML — fleet managers tune rules per vehicle class without touching code.

Deep Dive

Understand it at your level

Think of it as a co-pilot that speaks mechanic

Your vehicle has hundreds of sensors — temperature, speed, pressure, battery voltage — producing thousands of readings every minute. No human can watch all of them.

AnomEdge is a tiny brain that lives inside the vehicle (on a phone, tablet, or small computer). It watches every sensor simultaneously and knows what "normal" looks like for your specific vehicle.

When something goes wrong — maybe the engine is getting too hot, or the brakes are being slammed too often — it sends you a message on your phone in plain language:

🚨
Engine Overheating
"Pull over within 5 minutes. Turn off the A/C and turn the heater to full — it draws heat away from the engine. Check coolant level at your next safe stop."

No jargon. No cryptic codes. Just what's wrong and what to do about it. And because it runs on the vehicle itself, it works even in areas with no cell service.

Reduce downtime. Protect your drivers. Cut costs.

  • 18 configurable detection rules across thermal, braking, speed, hydraulic, transmission, electrical, fuel, and engine categories
  • Policy YAML — your ops team tunes thresholds per vehicle class (light truck vs. heavy equipment) without any code changes
  • 5 severity levels: NORMAL → WATCH → WARN → HIGH → CRITICAL, each with different driver-facing urgency
  • Cooldown & hysteresis prevent alert fatigue — the system won't nag about the same issue repeatedly
  • Offline-first — works in mines, rural areas, construction sites. No cloud dependency for critical alerts.
  • Voice alerts for HIGH and CRITICAL severity — the driver hears the alert even if they're not looking at their phone

ROI example: A fleet of 200 trucks averaging 2 breakdowns/month at $12K each = $4.8M/year. Catching 40% of those failures early (industry benchmark for predictive maintenance) saves $1.9M annually.

Built for performance, portability, and reliability

  • Core written in Rust — zero garbage collection pauses, memory-safe, compiles to WASM, iOS, Android, and IoT targets from a single codebase
  • Multi-target deployment: same inference logic runs in-browser (WASM), on phones (FFI), on IoT gateways (native ARM), and in the cloud (replay/training)
  • 3-tier inference with graceful degradation — deep learning, statistical ML, and expert rules in a cascading fallback chain
  • Pluggable adapters for different vehicle protocols — add new vehicle types without touching core logic
  • Noise-filtered alerting — intelligent suppression prevents duplicate/stale alerts from reaching the driver
  • All thresholds externalized — fleet managers adjust detection sensitivity without code changes or redeployment

Test coverage: 145 tests across unit, integration, and scenario-based testing. CI/CD via GitHub Actions.

Designed for zero single points of failure

MULTI-TARGET

Write Once, Run Everywhere

Single Rust codebase compiles to 5 targets: browser (WASM), iOS, Android, IoT gateways, and cloud. Same inference logic on every platform — no porting, no divergence.

GRACEFUL DEGRADATION

Cascading Fallback Chain

3-tier inference with automatic failover. If any tier is unavailable, times out, or produces low-confidence results, the next tier catches it. The system always produces a result.

PERFORMANCE

< 100ms p99 Pipeline Latency

Zero-GC Rust core with deterministic latency. No garbage collection pauses, no JIT warm-up. Consistent sub-100ms performance from raw signal to driver alert under load.

EXTENSIBILITY

Contract-First, Plugin Architecture

Frozen type contracts ensure cross-team stability. Pluggable vehicle adapters and externalized policy allow new vehicle types and rule sets without touching core code.

The architecture prioritizes safety over elegance. If any component fails — model corruption, runtime crash, hardware fault — the system falls back, logs the degradation, and still protects the driver. It never fails silently.
